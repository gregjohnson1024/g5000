# G5000 Plan 5 — NGT-1 Binary Protocol Support

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task.

**Goal:** Replace `Ngt1Driver`'s current Actisense-ASCII line parser with a real Actisense-binary frame parser so that a connected NGT-1 actually produces data through our pipeline. Today, the driver parses text in canboat's ASCII format — which is what `canboat`'s `actisense-serial` tool *produces* after decoding NGT-1 binary, not what the NGT-1 itself emits over USB. When you plug a real NGT-1 in, the current driver sees binary bytes and parses zero frames.

**Architecture:**
- Use canboatjs's `readN2KActisense` (in `lib/n2k-actisense.js`) — the same code path signalk and openplotter use in production.
- The driver buffers incoming bytes and detects complete Actisense packets by header (`0x10 0x02 0xd0`) + length field + footer (`0x10 0x03`). Each complete packet goes to `readN2KActisense` with `plainText: false`, which returns a structured `{ pgn: {pgn, src, dst, prio}, data: Buffer }` object. We rebuild a `RawCanFrame` from those fields and emit on `rxCan` as before — keeping the WireDriver contract unchanged.
- Test fixtures are generated by calling canboatjs's matching `encodeN2KActisense` encoder. The round-trip (encode → wire → decode) gives us a reliable in-memory smoke test that doesn't need hardware.
- `parseActisenseLine` (the existing ASCII text parser) stays exported as a *test utility* — it's still useful for constructing synthetic frames from hand-written ASCII in decoder.test.ts and channel-mapper.test.ts. It is no longer in the live wire path.

**Tech stack additions:** none. canboatjs is already installed.

**Reference spec:** `docs/superpowers/specs/2026-05-08-h6000-design.md` §4.4 — the wire-driver layer is the only thing changing here; the decoder + channel mapper + bus above it are untouched.

## What's NOT in scope

- Changing `decodeFrames` / channel mapper / TX wiring. All unchanged.
- Changing the wire contract (`WireDriver` interface stays the same).
- Multi-source CAN bus management (the device-claim / address-claim story is Plan 6).
- TX changes — the existing `txPgn` uses `pgnToActisenseSerialFormat` (ASCII), which is what the NGT-1 accepts for TX commands. RX and TX are asymmetric in Actisense's protocol design.

## Task 1: Switch `Ngt1Driver` to Actisense binary parsing (TDD)

**Files:**

The current driver's `onData(chunk)`:
1. Appends `chunk.toString('utf8')` to a string buffer
2. Splits on `\n`
3. Calls `parseActisenseLine` on each line
4. Emits the RawCanFrame

The new `onData(chunk)`:
1. Appends raw bytes to a Buffer
2. Looks for the next complete Actisense packet starting with `0x10 0x02 0xd0`
3. Reads the 2-byte little-endian length, waits until enough bytes are buffered
4. Verifies the trailing `0x10 0x03` ETX bytes
5. Calls `readN2KActisense` on the complete packet
6. Reconstructs a RawCanFrame from the callback's `{pgn, data}` and emits it
7. Advances the buffer past the consumed packet; loops if more remains

- [ ] **Step 1: Extend `vendor.d.ts` to declare the sub-module**

The current `vendor.d.ts` declares `module '@canboat/canboatjs'`. We need an additional declaration for the sub-path. Append:

```ts
declare module '@canboat/canboatjs/lib/n2k-actisense.js' {
  /**
   * Parse Actisense binary frames from a Buffer.
   * - `data` is one or more complete framed packets starting with `0x10 0x02 0xd0`.
   * - `plainText: false` causes the callback to receive { pgn, length, data, coalesced } objects.
   *   `pgn` is the result of canId-decode: { pgn, src, dst, prio }.
   * - The function processes packets in a tight loop until the buffer is exhausted
   *   or a partial packet is detected at the end (in which case it returns silently).
   */
  export function readN2KActisense(
    data: Buffer,
    plainText: boolean,
    context: unknown,
    cb: (result: {
      pgn: { pgn: number; src: number; dst: number; prio: number };
      length: number;
      data: Buffer;
      coalesced: boolean;
    }) => void,
  ): unknown;

  /**
   * Encode a PGN payload as an Actisense binary frame (Buffer). Inverse of
   * readN2KActisense. Used in tests to generate fixtures.
   */
  export function encodeN2KActisense(args: {
    pgn: number;
    data: Buffer;
    prio?: number;
    dst?: number;
    src?: number;
    timestamp?: number;
  }): Buffer;
}
```

The new flow:

```ts
import { Subject, type Observable, BehaviorSubject, EMPTY } from 'rxjs';
import canboat from '@canboat/canboatjs';
import { readN2KActisense } from '@canboat/canboatjs/lib/n2k-actisense.js';
import type {
  RawCanFrame,
  Raw0183Sentence,
  WireDriver,
  DriverHealth,
  OutgoingPgn,
} from './wire-driver.js';

const { pgnToActisenseSerialFormat } = canboat as unknown as {
  pgnToActisenseSerialFormat: (pgn: {
    pgn: number;
    prio?: number;
    dst?: number;
    src?: number;
    fields: Record<string, unknown>;
  }) => string;
};

/**
 * Anything that emits 'data' Buffer events. The serialport SerialPort class
 * matches this shape; a test harness can substitute a fake.
 */
export interface Ngt1Source {
  on(event: 'data', cb: (chunk: Buffer) => void): this;
  off(event: 'data', cb: (chunk: Buffer) => void): this;
}

export interface Ngt1Sink {
  write(
    buf: Buffer | string,
    cb?: (err?: Error | null) => void,
  ): boolean;
}

export interface Ngt1DriverOptions {
  source: Ngt1Source;
}

/**
 * Actisense binary framing constants (per canboatjs/lib/n2k-actisense.js).
 *
 *   0x10 0x02 0xd0 LL LL <payload (LL bytes)> 0x10 0x03
 *
 * The payload's last two bytes are the ETX marker (0x10 0x03). Total bytes
 * consumed per packet = 5 (header + length) + LL (payload incl. ETX).
 */
const STX_0 = 0x10;
const STX_1 = 0x02;
const STX_2 = 0xd0;
const HEADER_LEN = 5; // STX(3) + length(2)
const MIN_PACKET = HEADER_LEN + 2; // header + at least ETX

/**
 * NGT-1 driver. Reads Actisense binary frames from the serial source,
 * decodes them via canboatjs's readN2KActisense, and emits a RawCanFrame
 * per received PGN. ASCII text framing is NOT used at the wire level —
 * the parseActisenseLine helper remains exported for test-fixture use.
 */
export class Ngt1Driver implements WireDriver {
  readonly rxCan: Observable<RawCanFrame>;
  readonly rx0183: Observable<Raw0183Sentence> = EMPTY;
  readonly health: Observable<DriverHealth>;

  private readonly rxSubject = new Subject<RawCanFrame>();
  private readonly healthSubject = new BehaviorSubject<DriverHealth>({
    connected: false,
    bytesPerSecond: 0,
    framesPerSecond: 0,
    errorCount: 0,
  });
  private readonly source: Ngt1Source;
  private buffer: Buffer = Buffer.alloc(0);
  private dataHandler = this.onData.bind(this);

  constructor(opts: Ngt1DriverOptions) {
    this.source = opts.source;
    this.rxCan = this.rxSubject.asObservable();
    this.health = this.healthSubject.asObservable();
  }

  async start(): Promise<void> {
    this.source.on('data', this.dataHandler);
    this.healthSubject.next({ ...this.healthSubject.value, connected: true });
  }

  async stop(): Promise<void> {
    this.source.off('data', this.dataHandler);
    this.healthSubject.next({ ...this.healthSubject.value, connected: false });
  }

  async txCan(_frame: RawCanFrame): Promise<void> {
    throw new Error('Ngt1Driver.txCan not implemented (use txPgn)');
  }

  async tx0183(_port: number, _text: string): Promise<void> {
    throw new Error('Ngt1Driver.tx0183 not implemented (NGT-1 has no 0183)');
  }

  async txPgn(pgn: OutgoingPgn): Promise<void> {
    const line = pgnToActisenseSerialFormat({
      pgn: pgn.pgn,
      prio: pgn.prio ?? 6,
      dst: pgn.dst ?? 255,
      fields: pgn.fields,
    });
    if (!line || line.length === 0) {
      throw new Error(`canboatjs returned empty encoding for PGN ${pgn.pgn}`);
    }
    const sink = this.source as unknown as Ngt1Sink;
    if (typeof sink.write !== 'function') {
      throw new Error('Ngt1Driver.txPgn: source has no .write() method');
    }
    await new Promise<void>((resolve, reject) => {
      sink.write(line + '\n', (err) => (err ? reject(err) : resolve()));
    });
  }

  private onData(chunk: Buffer): void {
    this.buffer = this.buffer.length === 0 ? chunk : Buffer.concat([this.buffer, chunk]);
    while (true) {
      // Find the next start-of-frame marker.
      const start = this.findStart(this.buffer);
      if (start < 0) {
        // No header in buffer — keep at most the last 2 bytes (a partial header could span).
        if (this.buffer.length > 2) this.buffer = this.buffer.subarray(-2);
        return;
      }
      // Drop any pre-header garbage.
      if (start > 0) this.buffer = this.buffer.subarray(start);
      if (this.buffer.length < HEADER_LEN) return; // need the length field
      const payloadLen = this.buffer.readUInt16LE(3);
      const totalLen = HEADER_LEN + payloadLen;
      if (this.buffer.length < totalLen) return; // incomplete packet
      const packet = this.buffer.subarray(0, totalLen);
      this.buffer = this.buffer.subarray(totalLen);
      try {
        readN2KActisense(packet, false, {}, (result) => {
          this.emitFrame(result);
        });
      } catch (_err) {
        const h = this.healthSubject.value;
        this.healthSubject.next({ ...h, errorCount: h.errorCount + 1 });
      }
    }
  }

  private findStart(buf: Buffer): number {
    for (let i = 0; i + 2 < buf.length; i++) {
      if (buf[i] === STX_0 && buf[i + 1] === STX_1 && buf[i + 2] === STX_2) {
        return i;
      }
    }
    return -1;
  }

  private emitFrame(result: {
    pgn: { pgn: number; src: number; dst: number; prio: number };
    data: Buffer;
  }): void {
    const id = encodeJ1939Id(result.pgn.prio, result.pgn.pgn, result.pgn.src);
    this.rxSubject.next({
      id,
      ext: true,
      data: new Uint8Array(result.data),
      rxTimestamp: BigInt(Date.now()) * 1_000_000n,
    });
  }
}

// parseActisenseLine + encodeJ1939Id stay exactly as they are in the
// current file — both still useful. parseActisenseLine for test fixtures,
// encodeJ1939Id internal to emitFrame above.
//
// Keep the existing implementations below unchanged.
```

**Important**: the existing `encodeJ1939Id` function and `parseActisenseLine` function are still needed in this file. `encodeJ1939Id` is called by `emitFrame` above (it packs the canId we got from canboatjs's parser back into a 29-bit ID for our RawCanFrame contract). `parseActisenseLine` stays exported because `decoder.test.ts` still uses it to construct test fixtures from hand-written ASCII. Keep both functions unchanged at the bottom of the file — only `onData`, the constructor's buffer field type (`string` → `Buffer`), and the imports change.
